Quickstart

From zero to a confirmed booking in three steps. Grab a key, install an SDK, make a call.

1
Create an API key
In your dashboard under Settings → Developers. Test keys start with sk_test_, live keys with sk_live_.
2
Install the SDK
npm install @roompanda/sdk
3
Make your first call
Fetch availability, then POST a booking. That's the whole loop — see the reference below for every field.

Authentication

Every request is authenticated with a bearer token. Keep secret keys server-side; scope tokens per property and per capability so a leaked key can only do what you allowed.

Authorization: Bearer sk_live_panda
Content-Type:  application/json

API reference

Predictable, resource-oriented endpoints with JSON bodies and standard HTTP codes. The core six:

GET /v1/properties List your properties & settings
GET /v1/availability Bookable rooms & rates for dates
GET /v1/rates Rate plans & restrictions
POST /v1/bookings Create a confirmed booking
GET /v1/bookings/:id Retrieve a booking
POST /v1/bookings/:id/cancel Cancel & release inventory
Request
POST /v1/bookings
{
  "property": "the-laurel",
  "room":     "garden-view-king",
  "rate":     "flexible",
  "checkin":  "2026-09-12",
  "nights":   3,
  "guest": { "name": "Jamie Doyle" }
}
Response · 201
{

  "currency": "EUR",
  "cancel_by": "2026-09-11T15:00Z"
}
Model Context Protocol

MCP server

Point any MCP-compatible AI client at Roompanda and it can search, price and book on a guest's behalf — using the same scoped tokens and rate fences as the REST API. Bookings are real and land straight in your calendar.

Connect a client
{
  "mcpServers": {
    "roompanda": {
      "url": "https://mcp.roompanda.com/sse",
      "headers": {
        "Authorization": "Bearer sk_live_panda"
      }
    }
  }
}
Exposed tools
search_availability
Find bookable rooms for dates & guests.
get_room
Full detail, photos & rate plans for a room.
book_room
Place a real, confirmed reservation.
cancel_booking
Cancel within your cancellation policy.
Agents only see what your token allows. Set per-tool permissions, maximum nightly rates and an optional human-approval step for bookings over a threshold.

Webhooks

Subscribe to events and we'll POST a signed payload to your endpoint the moment anything changes — no polling required.

booking.created
booking.modified
booking.cancelled
payment.succeeded
POST  https://your-app.com/hooks
