Quickstart

From zero to a booking in three steps — grab a key, check availability, POST a booking.

1
Create an API key
In your admin under API keys. Test keys start with sk_test_, live keys with sk_live_.
2
Check availability
curl https://book.roompanda.com/v1/availability\
  ?checkin=2026-09-12&checkout=2026-09-15&adults=2 \
  -H "Authorization: Bearer sk_test_2b1c…"
3
Make your first call
Fetch availability, then POST a booking. That's the whole loop — see the reference below for every field.

Authentication

Every request carries a bearer key. Keys are scoped to a single property and created in your admin under API keys. Test keys (sk_test_) create simulated bookings; live keys (sk_live_) create real ones. The key is shown once — store it server-side; revoke it any time.

Authorization: Bearer sk_test_2b1c…
Content-Type:  application/json

API reference

Predictable, resource-oriented endpoints with JSON bodies and standard HTTP codes. Base URL https://book.roompanda.com. Each key is scoped to one property, so reads need no property id.

GET /v1/properties Your property & its details
GET /v1/calendar Per-date open/closed & min-stay (date picker)
GET /v1/rooms Room content, photos & facilities
GET /v1/availability Priced, bookable rooms & rates for a stay
GET /v1/rates Rate plans & cancellation policies
GET /v1/extras Add-ons / upsells catalogue
POST /v1/bookings Create a booking (returns a payment link if a deposit is due)
GET /v1/bookings List bookings
GET /v1/bookings/:id Retrieve a booking

Errors return { "error": { "type", "message" } } with the matching HTTP status. Send an Idempotency-Key header on POST /v1/bookings to make retries safe.

{
  "checkin":  "2026-09-12",
  "checkout": "2026-09-15",

MCP server

Built on the same REST API: point any MCP-compatible AI client at Roompanda and it can search, price and book on a guest's behalf — using the same per-property keys and policy fences. Bookings are real and land straight in your calendar. Source is on GitHub (roompanda-mcp ↗); the npm package below lands shortly.

Connect a client
{
  "mcpServers": {
    "roompanda": {
      "url": "https://mcp.roompanda.com/sse",
      "headers": {
        "Authorization": "Bearer sk_live_2b1c…"
      }
    }
  }
}

Webhooks

Subscribe to events and we'll POST a signed payload to your endpoint the moment anything changes — no polling required.

booking.created
booking.cancelled

Add an endpoint & pick events in your admin under Webhooks. Each endpoint gets its own signing secret (whsec_…).

POST  https://your-app.com/hooks
Roompanda-Signature: t=1789…,v1=9266…

{
  "id":      "evt_3f9c…",
  "type":    "booking.created",
  "created": 1789000000,
  "data": {
    "reference": "9Z8MSVF1",
    "status":    "confirmed",
    "total":     420,
    "currency":  "GBP"
  }
}

Verify: v1 == HMAC-SHA256(secret, t + "." + body).
